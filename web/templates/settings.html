Add MEGA account

    Master password

    Encrypts stored account credentials at rest with a password only you know. If you forget it, saved accounts can't be recovered -- you'd need to remove and re-add them.

    Not yet implemented

    • ELC / MegaCrypter accounts (third-party link decryption service is out of scope for now).
    • Choosing which saved account handles anonymous public-link downloads -- downloads are always anonymous right now, never using a logged-in session's quota.

    Downloads

    Not yet implemented

    • Bandwidth speed limiting for downloads (no throttling layer built yet).
    • Custom directory for keeping partial/temp download data (this port doesn't write side temp files -- chunks are reassembled in memory as they arrive).

    Uploads

    Not yet implemented

    • Bandwidth speed limiting for uploads.
    • Auto-generating and attaching a thumbnail when uploading (the thumbnailer itself exists and works standalone -- it just isn't wired into the upload pipeline's MEGA thumbnail-attach API yet).
    • Auto-making uploaded folders public / sharing links automatically.
    • Always reloading the account's MEGA folder tree before upload (no folder-tree caching for your own account exists yet -- only public folder links can be browsed).
    • Saving an upload log file.
    • Custom directory for partial upload chunk data (uploads stream directly from source to network, no chunk staging directory involved).

    SmartProxy

    Proxy list, one per line: ip:port, *ip:port for SOCKS, ip:port@base64user:base64pass, or #https://url-to-a-list

    Not yet implemented

    • "Reset slot to direct connection after a successful proxied request" -- accepted as a concept in the code but not yet wired to actual per-worker routing decisions.
    • Auto-refresh proxy list on a timer (only manual refresh / refresh-on-save exist right now).
    • Quota-recovery IP-change polling (auto-resume when your public IP changes, e.g. after a VPN reconnect).
    • Stall timeout and post-509 recheck window tuning.

    Advanced

    Saves RAM at a high number of slots and concurrent downloads by staging chunks to disk instead of holding them in memory, at the cost of doubling disk I/O.

    Not yet implemented

    • Manual single proxy (host/port/user/pass) -- use the SmartProxy list with one entry + Force mode for similar behavior.
    • External command hooks (509-recovery script, post-download-queue-finish, post-upload-queue-finish).
    • MegaCrypter reverse proxy.
    • Starting with the transfer queue frozen -- moot for now anyway, since in-progress transfers don't persist across an app restart yet (they're tracked in memory only).
    • Language / font / UI zoom (this port is English-only; use your browser's own zoom).
    • Clipboard link auto-detection -- a browser page can't silently poll the OS clipboard the way a desktop app can. Paste links directly into the download field instead.
